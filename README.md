# 쇼츠 처형자 (Shorts Executioner)

양산형·저작권 무시·날먹 유튜브 쇼츠/추천 영상을 정리하는 브라우저 확장 프로그램입니다. 두 가지 무기를 제공합니다.

### 1. 원클릭 "처리" 버튼 🚫
한 번 클릭으로 그 영상에:
- 👎 **싫어요** (쇼츠 한정 — 피드 썸네일엔 싫어요 버튼이 없음)
- 🙈 **관심 없음**
- 🚫 **채널 추천 안함**

을 순서대로 자동으로 누릅니다. (한 카드에서 관심없음/채널추천안함은 유튜브 특성상 1개만 적용됨 — 아래 [한계](#동작-방식--한계) 참고)

### 2. "채널 영구 차단" 버튼 🚷
누른 채널을 **확장 자체 차단 목록**에 넣고, 그 채널을 다시는 안 보이게 합니다.
- **추천/검색/관련영상에서 렌더 전에 제거** — 데이터(Innertube JSON)를 가로채 차단 채널 항목을 잘라내므로, 목록에 **뜨기도 전에** 사라집니다 (BlockTube식).
- **쇼츠 자동 스킵** — 차단 채널의 쇼츠가 나오면 **재생 전에** 다음으로 넘깁니다 (재생 정지 + 다음 이동).
- **피드 DOM 숨김** — 인터셉터가 놓친 잔여 카드는 추가로 숨깁니다 (안전망).

> 신고 기능은 **일부러 넣지 않았습니다.** 자동·대량 신고는 유튜브 신고 기능 남용(약관 위반) 소지가 있고 역효과가 날 수 있습니다. 자세한 이유는 [정책 메모](#정책-메모-왜-신고는-없나요) 참고.

## 설치 (개발자 모드 / Load unpacked)

### Chrome · Edge · Brave 등 (Chromium)
1. `chrome://extensions` (Edge는 `edge://extensions`)
2. 우측 상단 **개발자 모드** 켜기
3. **압축해제된 확장 프로그램을 로드합니다** → 이 폴더(`executioner`) 선택
4. 유튜브 새로고침

### Firefox
- `world: "MAIN"` 콘텐츠 스크립트(인터셉터)를 쓰기 때문에 **Firefox 128 이상** 필요.
- `about:debugging#/runtime/this-firefox` → **임시 부가 기능 로드** → 이 폴더의 `manifest.json` 선택.
- (임시 로드는 브라우저를 닫으면 사라집니다. 영구 설치는 AMO 서명 필요.)

## 사용법

- **쇼츠**: 우측 버튼 모음 위에 🚫(처리)·🚷(차단) 버튼이 생깁니다.
- **추천/검색/관련영상**: 썸네일에 마우스를 올리면 우측 상단에 두 버튼이 나타납니다.
- 차단 버튼을 누르면 그 채널이 차단 목록에 들어가고, 이후 그 채널은 추천/검색에 안 뜹니다. 쇼츠면 즉시 다음으로 넘어갑니다.
- 처리/차단되면 화면 하단에 토스트가 잠깐 뜹니다.

### 설정 팝업 (툴바 아이콘)

| 설정 | 설명 |
|---|---|
| 확장 켜기 | 전체 on/off |
| 쇼츠 플레이어 / 추천·피드 썸네일 | 버튼 표시 위치 |
| 싫어요도 함께 | 쇼츠에서 싫어요까지 누름 |
| 메뉴 동작 | `둘 다 시도` / `채널 추천 안함만` / `관심 없음만` |
| 채널 차단 버튼 | 차단 버튼 표시 on/off |
| 차단 쇼츠 자동 스킵 | 차단 채널 쇼츠를 재생 전 스킵 |
| 피드에서 숨김 (DOM 안전망) | 인터셉터가 놓친 카드 추가 숨김 |
| 차단된 채널 | 목록 보기 / 개별·전체 차단 해제 |
| 동작 간 지연(ms) | 너무 빠르면 유튜브 `금지된 작업입니다` 제한에 걸림 |

## 동작 방식 (아키텍처)

```
[MAIN 월드] src/intercept.js   ← document_start, world:"MAIN"
   · window.fetch + ytInitialData 후킹
   · Innertube JSON 재귀 순회 → 차단 채널 renderer를 splice (렌더 전 제거)
   · 차단목록은 localStorage 미러를 동기로 읽고, 갱신은 postMessage로 수신
        ▲ (localStorage 미러 + window.postMessage 브리지)
        │
[격리 월드] config / util / blocklist / youtube / content.js  ← document_idle
   · 버튼 주입, 채널 식별자 추출(@handle/UC…/이름)
   · 차단목록 저장(chrome.storage.local) + 미러링
   · DOM 안전망(피드 숨김) + 쇼츠 자동 스킵
[팝업] popup/*  ← 설정(sync) + 차단목록(local) 관리
```

**왜 이렇게?**
- 추천/검색에 "뜨기도 전에" 없애려면 렌더 전 데이터를 잘라야 하고, 그러려면 YouTube가 실제로 쓰는 `window.fetch`/`ytInitialData`를 덮어써야 합니다. 이는 페이지(MAIN) 월드에서만 가능합니다.
- 인라인 `<script>` 주입은 YouTube의 Trusted Types CSP에 막히므로, `content_scripts`의 `world:"MAIN"`으로 주입해 페이지 CSP를 우회합니다.
- MAIN 월드는 `chrome.storage`를 못 읽으므로, 차단목록을 `localStorage`에 미러링해 인터셉터가 `document_start`에 동기로 즉시 읽습니다. 세션 중 변경은 `postMessage`로 푸시합니다.

## 동작 방식 & 한계

- **한 카드에서 "관심 없음"과 "채널 추천 안함"을 동시에 둘 다는 못 누릅니다.** 유튜브가 메뉴 항목 하나를 누르는 순간 카드를 "실행취소" 카드로 바꿔 두 번째 대상이 사라지기 때문입니다. 기본값은 더 강한 신호인 **채널 추천 안함 우선**입니다.
- **채널 식별자**: DOM은 보통 변하기 쉬운 `@handle`만 노출합니다. 인터셉터는 JSON에서 불변의 `UC…` ID를 읽어 매칭하고, 차단 매칭은 **ID·핸들·이름 중 무엇이든** 되도록 했습니다.
- **쇼츠 셸프(피드 안의 쇼츠 묶음)**: `shortsLockupViewModel` JSON에는 채널 ID가 없어 인터셉터로 채널 필터가 어렵습니다. 쇼츠는 **플레이어 자동 스킵**이 주 방어선입니다.
- **차단 해제**는 이미 로드된 페이지에서는 새로고침 후 완전 반영됩니다 (인터셉터가 이미 잘라낸 항목은 그 페이지에선 복구되지 않음).
- **셀렉터 견고성**: 깨지기 쉬운 클래스/위치 대신 id·태그·다국어 텍스트·href 규칙으로 찾습니다. 유튜브 개편으로 못 찾으면 토스트로 알려 줍니다. 메뉴 언어가 다르면 `src/config.js`의 `LABELS`에 추가하세요.

## 개인정보
- 외부로 **아무 데이터도 보내지 않습니다.** 네트워크 요청을 새로 만들지 않고, 기존 응답을 가로채 필터링만 합니다.
- 설정은 `chrome.storage.sync`, 차단 목록은 `chrome.storage.local`(+ youtube.com `localStorage` 미러)에 저장됩니다.
- 권한은 `storage` 와 `youtube.com` 콘텐츠 스크립트뿐입니다.

## 정책 메모: 왜 신고는 없나요?
조사 결과 요약 (2025–2026):
- **"저품질 AI/TTS 날먹" 자체는 삭제 사유가 아닙니다.** 수익화(YPP)는 2025-07-15부터 *비정품·대량생산 콘텐츠*를 수익화에서 배제(=돈만 끊김, 영상은 남음). 삭제·경고는 *스팸·조작·사기·탐지 회피* 영역으로 넘어갈 때만.
- **저작권**은 강력하지만 권리자 본인만 별도 웹폼으로 신고 가능 (제3자 시청자는 불가).
- **자동·대량 신고는 신고 기능 남용**으로 약관 위반 소지 + 역풍 위험.

→ 그래서 효과적이고 안전한 수단인 **싫어요·관심없음·채널추천안함 + 자체 차단 목록**만 자동화했습니다.

## 폴더 구조
```
executioner/
├─ manifest.json
├─ src/
│  ├─ intercept.js   # MAIN 월드: Innertube JSON 가로채기(렌더 전 차단)
│  ├─ config.js      # 기본 설정 · 다국어 라벨 · 셀렉터
│  ├─ util.js        # waitFor / 클릭 / 호버 / 토스트
│  ├─ blocklist.js   # 차단 목록 저장 + 조회 + compile
│  ├─ youtube.js     # 채널 추출 + 메뉴 동작 + 쇼츠 내비게이션
│  ├─ content.js     # 버튼 주입 + 강제(숨김/스킵) + 브리지
│  └─ styles.css     # 버튼/토스트/숨김 스타일
├─ popup/            # 설정 + 차단 목록 관리 UI
└─ icons/            # 아이콘 + 생성 스크립트
```
